# GigaStudy 작업 원칙

기준일: 2026-04-07

이 문서는 GigaStudy 저장소에서 작업할 때 반드시 따르는 운영 원칙을 고정한다.

이 원칙은 설계 문서보다 아래에 있는 팁이 아니라, 실제 구현·정리·빌드·배포 방식의 기준이다.

## 1. 버전 관리는 커밋으로 한다

- 작업 이력 관리는 로컬에 파일을 쌓아 두는 방식이 아니라 `git commit`을 기준으로 한다.
- `temp`, `backup`, `copy`, `final_final`, 날짜별 임시 파일 같은 로컬 중복본을 남기지 않는다.
- 작업 중간 상태를 보존해야 하면 임시 파일이 아니라 명확한 커밋 단위로 남긴다.
- 저장소 안에는 현재 기준에서 필요한 파일만 둔다.

## 2. 프로젝트 구조는 평평하고 응집되게 유지한다

- 폴더를 수평적으로 무한정 늘리거나, 깊이를 불필요하게 중첩하는 구조를 지양한다.
- 새 폴더는 실제 책임이 분명하고, 기존 폴더로 흡수할 수 없을 때만 만든다.
- 새 파일도 같은 기준을 따른다.
- 기능 추가보다 폴더 구조 증가가 더 빨라지면 구조를 다시 정리한다.

## 3. 테스트 파일과 중간 산출물은 바로 정리한다

- 테스트용 데이터, 임시 리포트, 검증 스냅샷, 빌드 부산물, 캐시 파일은 작업 후 바로 정리한다.
- 정말 유지할 가치가 있는 산출물만 구조화된 공식 폴더에 분류해서 남긴다.
- 테스트 파일도 장기적으로 유지할 기준이 분명할 때만 남긴다.
- 검증 때문에 잠깐 만든 의존성 폴더, 캐시, 산출물은 검증이 끝나면 제거한다.

## 4. 적용 방식

- 설계, 코드, 테스트, 빌드 스크립트가 이 원칙과 충돌하면 원칙에 맞게 정리한다.
- 작업 도중 임시 예외가 필요하더라도, 턴이 끝날 때는 다시 이 원칙에 맞는 상태로 되돌린다.
- 새 기준 문서를 추가할 때도 이 원칙을 먼저 확인한다.
