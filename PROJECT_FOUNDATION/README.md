# Project Foundation

기준일: 2026-04-07

이 폴더는 GigaStudy의 제품 기준, 구현 순서, 출시 판정 기준을 모아 둔 공식 루트다.

여기 문서는 단순 참고용 메모가 아니라, 실제 개발 우선순위와 완료 기준을 고정하는 기준 문서로 다룬다.

## 읽는 순서

1. `GigaStudy_master_plan.md`
2. `ROADMAP.md`
3. `PHASE1_BACKLOG.md`
4. `GigaStudy_check_list.md`
5. `WORKING_PRINCIPLES.md`

## 문서 역할

- `GigaStudy_master_plan.md`
  제품 정의, MVP 범위, 기술 스택, 아키텍처, 리스크, 출시 컷라인을 정리한다.
- `ROADMAP.md`
  구현 단계를 어떤 순서로 닫을지, 각 단계의 산출물과 완료 기준이 무엇인지 정리한다.
- `PHASE1_BACKLOG.md`
  첫 구현 단계인 녹음 파이프라인과 DeviceProfile 구축을 작업 티켓 수준으로 분해한다.
- `GigaStudy_check_list.md`
  착수 전 확인, 구현 중 점검, 출시 직전 검수를 한 번에 확인할 수 있는 실행 체크리스트다.
- `WORKING_PRINCIPLES.md`
  저장소 운영 원칙, 파일 구조 기준, 임시 산출물 정리 기준을 고정한다.

## 운영 원칙

- 이 폴더의 문서는 GigaStudy의 최상위 기준 문서로 취급한다.
- 코드, 백로그, UI 흐름이 기준 문서와 충돌하면 같은 작업 맥락 안에서 함께 수정한다.
- 새 기능을 추가할 때 범위, 구현 순서, 완료 기준이 바뀌면 이 폴더 문서를 먼저 갱신한다.
- 작업 방식 자체를 바꾸는 기준은 `WORKING_PRINCIPLES.md`에 먼저 반영한다.
